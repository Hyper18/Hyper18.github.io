数据结构篇：常用查找算法回顾

发表于 2022-06-29 更新于 2024-09-05 分类于 笔记 ， 408 ， 数据结构 阅读次数：

不同数据结构常用查找算法，譬如顺序查找、折半查找（二分）、分块查找、二叉排序树的查找、二叉平衡树的查找、B树的查找、散列表查找……

（一）基本概念

1. 查找

  • def 在数据集合中寻找满满足某种条件的数据元素的过程

2. 查找表

  • def 用于查找的数据集合，由同一类型数据元素组成

  • 操作

    • 查（是否有某元素、特定数据结构的某属性）
    • 插（同类型某数据元素）
    • 删（同类型某数据元素）

  • 分类

    • 静态查找表：顺序查找、折半查找和散列查找
    • 动态查找表：二叉排序树（二叉平衡树、B 树）的查找、散列查找

3. 关键字

  • def 数据元素中唯一标识该元素的某数据项的值

  • method 使用关键字查找，结果唯一

4. 平均查找长度 average search length

  • def 所有查找过程中进行进行关键字比较次数的平均值

    • quiz 衡量查找算法效率用什么指标？ASL

  • 公式 令 n 为查找表长度，Pi=1nP_i = \frac{1}{n}Pi​=n1​为查找第 i 个数据元素的概率，cic_ici​为找到第 i 个数据元素的比较次数

    ASL=∑i=1nPiCiASL = \sum_{i=1}^{n}{P_iC_i} ASL=i=1∑n​Pi​Ci​

（二）线性表查找

1. 顺序查找 / 线性查找

  • method 对含 n 个元素的表，定位第 i 个元素，需比较 n-i+1 次（注意为从后往前，严蔚敏版）

    ASLsuccess=∑i=1nPi(n−i+1)=n+12(Pi=1n 时)ASL_{success} = \sum_{i=1}^{n} P_i (n-i+1) = \frac {n+1}{2}(P_i = \frac {1}{n} 时) ASLsuccess​=i=1∑n​Pi​(n−i+1)=2n+1​(Pi​=n1​时)

    ASLfail=n+1ASL_{fail} = n + 1 ASLfail​=n+1

  • 优点 对数据元素的存储无要求（顺序、链式均可）

  • 缺点 n 较大 - ASL 较大，效率低

  • 时间复杂度 O(n)O(n)O(n)

  • 优化

    • quiz1 针对有序表的情况，有什么区别？

    ASLsuccess=∑i=1nPi(n−i+1)=n+12(Pi=1n 时)ASL_{success} = \sum_{i=1}^{n} P_i (n-i+1) = \frac {n+1}{2}(P_i = \frac {1}{n} 时) ASLsuccess​=i=1∑n​Pi​(n−i+1)=2n+1​(Pi​=n1​时)

    qj=1n+1q_j = \frac{1}{n+1}qj​=n+11​ 为到达第 j 个失败结点的概率，ljl_jlj​为第 j 个失败结点所在的层数，则 ASL 可优化为：

    ASLfail=∑j=1nqj(lj−1)=n2+nn+1ASL_{fail} = \sum_{j=1}^{n}q_j(l_j-1) = \frac{n}{2} + \frac{n}{n+1} ASLfail​=j=1∑n​qj​(lj​−1)=2n​+n+1n​

    • quiz2 针对被查概率不相等的情况，又有什么方法？

      答：可以将被查概率大的数据元素前置，从降低关键字的比较次数 CiC_iCi​入手，降低 ASL

2. 折半查找 / 二分

  • method 每次找中间位置比较，若与给定 key 相等则成功；大于给定 key 值，则应该在前半部分（升序排列时），反之亦然

    • 前提 仅适用于有序的顺序表

    ASLsuccess=1n∑i=1nli=n+1nlog2(n+1)−1≈log2(n+1)−1ASL_{success} = \frac{1}{n}\sum_{i=1}^{n}l_i = \frac{n+1}{n}log_2(n+1)-1 \approx log_2(n+1)-1 ASLsuccess​=n1​i=1∑n​li​=nn+1​log2​(n+1)−1≈log2​(n+1)−1

  • def 判定树：折半查找形成的二叉树，是一棵平衡二叉树

    树高 h=⌈log2(n+1)⌉树高 h = \lceil log_2 (n+1) \rceil 树高 h=⌈log2​(n+1)⌉

  • 优点 时间复杂度较降低，为 O(log2n)O(log_2{n})O(log2​n)

  • 缺点 线性表需能随机存储，即仅顺序存储结构适用，且需有序

3. 分块查找

  • method 在索引表中确定待查记录所属的分块（可顺序、可折半），在块内顺序查找

    • 特点 块内无序 块间有序
    • 假设长 n 的查找表被均匀分为 b 块，每块 s 个元素，设索引查找和块内查找的平均查找长度分别为 LIL_ILI​、LSL_SLS​，则分块查找的平均查找长度为 ASL=LI+LSASL=L_I + L_SASL=LI​+LS​
      • 顺序查找：LI=1+2+...+bb=b+12,LS=s+1sL_I = \frac{1+2+...+b}{b} = \frac{b+1}{2},L_S = \frac{s+1}{s}LI​=b1+2+...+b​=2b+1​,LS​=ss+1​；ASL=s2+2s+n2sASL=\frac{s^2+2s+n}{2s}ASL=2ss2+2s+n​，当 s=ns = \sqrt{n}s=n​时，ASLmin=n+1ASL_{min} = \sqrt{n+1}ASLmin​=n+1​
      • 折半查找：LI=⌈log2(b+1)⌉,LS=s+1sL_I =\lceil log_2(b+1) \rceil,L_S = \frac{s+1}{s}LI​=⌈log2​(b+1)⌉,LS​=ss+1​；ASL=⌈log2(b+1)⌉+s+12ASL=\lceil log_2(b+1) \rceil + \frac{s+1}{2}ASL=⌈log2​(b+1)⌉+2s+1​

（三）树表查找

1. 二叉排序树 BST

  • val 左 ＜ 根 ＜ 右

  • 时间复杂度 O(log2n)O(log_2n)O(log2​n)：n 个结点的二叉树最小高度为⌊log2n⌋+1\lfloor log_2n \rfloor+1⌊log2​n⌋+1，时间复杂度最好 O(log2n)O(log_2n)O(log2​n)，最坏 O(n)O(n)O(n)

2. 平衡二叉树 AVL

  • def 树上任一结点的左子树和右子树的高度之差不超过 1

  • 假设以 nhn_hnh​表示深度为 h 的平衡树中含有的最少结点数， 则 n0=0,n1=1,n2=2n_0 = 0, n_1 = 1, n_2 = 2n0​=0,n1​=1,n2​=2，且 nh=nh−1+nh−2+1n_h = n_{h−1} + n_{h−2} + 1nh​=nh−1​+nh−2​+1

  • 时间复杂度 O(log2n)O(log_2n)O(log2​n)

3. 多路平衡查找树 B 树

  • def 一棵 m 阶 B 树是所有结点孩子个数的最大值为 m 的多路平衡查找树

    • m 叉查找树中，除根节点的其他结点至少有⌈m/2⌉个分叉，⌈m/2⌉ − 1 个关键字
    • 对任意结点，其所有子树高度均相同

  • method m 叉查找树

  • 高度（不含叶结点，此时搜索失败） logm(n+1)≤h≤log⌈m/2⌉n+12+1log_m(n + 1) ≤ h ≤ log_{⌈m/2⌉}\frac{ n + 1 }{2} + 1logm​(n+1)≤h≤log⌈m/2⌉​2n+1​+1

B + 树

  • def 结点的子树个数 = 关键字个数，分支结点值 = 子树中关键字的最大值，相邻叶结点按大小链接

  • method 多级分块查找

  • 支持顺序查找，查找稳定

（四）散列表

  • def

    • 同义词：关键字通过散列函数映射到同⼀个值
    • 冲突：散列函数确定的位置已存放其他元素；降低冲突，才能提高查找效率
    • 装填因子：α= 表中记录数散列表长度 α = \frac {表中记录数}{散列表长度}α= 散列表长度表中记录数​

  • 时间复杂度：同时取决于散列函数、处理策略和装填因子，最好情况下复杂度为 O(1)O(1)O(1)

  • 散列函数

    • 除留余数法：H(key)=key%pH(key) = key \% pH(key)=key%p（p 是不大于散列表长的质数）
    • 直接定址法：H(key)=keyH(key) = keyH(key)=key / H(key)=a∗key+bH(key) = a*key + bH(key)=a∗key+b
    • 数字分析法：取数码分布较均匀的若干位
    • 平方取中法：取关键字平方值的中间几位
